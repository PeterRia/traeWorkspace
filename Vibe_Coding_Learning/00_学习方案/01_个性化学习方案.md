# 个性化 Vibe Coding 学习方案

## 学习目标

### 总体目标
通过系统化的 Vibe Coding 学习，从零基础成长为能够独立完成 AI 产品开发的开发者。

### 阶段目标

#### Stage 1 - 零基础入门（第 1-2 周）
- [ ] 理解 Vibe Coding 核心理念和 AI 编程能力边界
- [ ] 使用 z.ai 等平台独立完成 AI 原生小游戏
- [ ] 掌握需求挖掘方法论，找到值得做的产品方向
- [ ] 完成从想法到可交互原型的完整闭环

#### Stage 2 - 初中级开发（第 3-5 周）
- [ ] 学会使用 Lovart/Nanobanana 搭建素材工作流
- [ ] 掌握 Figma/MasterGo 基础操作和设计协作
- [ ] 实现设计稿到前端代码的转换
- [ ] 使用 Supabase 构建真实数据项目
- [ ] 完成产品从本地到云端的部署上线
- [ ] 集成 Dify 构建 AI 知识库应用

#### Stage 3 - 高级开发（第 6-9 周）
- [ ] 使用 Electron 开发跨平台桌面应用
- [ ] 掌握 Claude Agent Teams 多代理协作
- [ ] 学会管理长时间运行的稳定任务
- [ ] 搭建个人品牌网站与技术博客

---

## 进度安排

### 每日学习建议
- **工作日**：每天 1-2 小时（理论 + 实践）
- **周末**：每天 3-4 小时（项目实战为主）
- **灵活调整**：根据个人节奏动态调整

### 周进度规划

| 周次 | 阶段 | 学习内容 | 重点 | 产出 |
|:---:|------|---------|------|------|
| 1 | Stage1 | 学习地图 + 游戏化入门 | 建立 AI 编程手感 | AI 原生贪吃蛇 |
| 2 | Stage1 | 产品原型实战 | 需求挖掘与验证 | 可交互原型 |
| 3 | Stage2 | 素材生成 + UI 设计 | 设计工具入门 | 设计稿 |
| 4 | Stage2 | 设计转代码 + 数据库 | 前后端衔接 | 数据驱动应用 |
| 5 | Stage2 | 部署上线 + AI 集成 | 全栈闭环 | 上线产品 |
| 6 | Stage3 | 跨平台桌面应用 | Electron 基础 | 桌面应用 |
| 7 | Stage3 | AI 智能体团队 | 多 Agent 协作 | Agent 项目 |
| 8 | Stage3 | 长效执行 + 个人品牌 | 稳定运行 | 个人网站 |
| 9 | 综合 | 项目整合与优化 | 综合实战 | 完整作品集 |

## 重点难点提示

### Stage 1 重点难点
| 知识点 | 难度 | 关键突破点 |
|--------|:--:|-----------|
| AI 能力边界理解 | ⭐⭐ | 多实践、多对比不同 AI 工具的输出 |
| 提示词设计 | ⭐⭐⭐ | 学习"把需求说清楚"，参考模板库 |
| 需求验证 | ⭐⭐⭐⭐ | 必须与真实用户对话，不可凭空想象 |
| 产品原型开发 | ⭐⭐⭐ | 从简单页面开始，逐步增加复杂度 |

### Stage 2 重点难点
| 知识点 | 难度 | 关键突破点 |
|--------|:--:|-----------|
| 设计工具操作 | ⭐⭐⭐ | 先模仿再创造，多看优秀设计 |
| 数据库设计 | ⭐⭐⭐⭐ | 从单表开始，理解关系型数据模型 |
| 部署流程 | ⭐⭐⭐ | 首次部署会遇坑，耐心排查 |
| RAG 应用 | ⭐⭐⭐⭐⭐ | 先理解检索增强的基本原理 |

### Stage 3 重点难点
| 知识点 | 难度 | 关键突破点 |
|--------|:--:|-----------|
| Electron 开发 | ⭐⭐⭐⭐ | 理解主进程与渲染进程的通信 |
| 多 Agent 编排 | ⭐⭐⭐⭐⭐ | 从双 Agent 简单协作开始 |
| 长任务稳定性 | ⭐⭐⭐⭐ | 做好错误处理和断点续传 |

## 能力评估标准

### 自评量表（每阶段完成后填写）

| 能力维度 | 入门(1分) | 基础(2分) | 熟练(3分) | 精通(4分) |
|---------|:--------:|:--------:|:--------:|:--------:|
| AI 工具使用 | 会基本对话 | 能引导 AI 输出 | 高效协作开发 | 搭建 Agent 团队 |
| 产品思维 | 理解需求概念 | 能验证需求真伪 | 独立设计产品 | 商业模式创新 |
| 前端开发 | 看懂 HTML | 能改样式布局 | 独立做页面 | 复杂交互实现 |
| 后端开发 | 了解 API | 会用数据库 | 设计数据模型 | 架构设计 |
| 部署运维 | 知道概念 | 会一键部署 | 配置域名 SSL | CI/CD 自动化 |
| AI 集成 | 调用 API | 搭建知识库 | RAG 应用 | Agent 编排 |

### 评分标准
- **Stage 1 完成标准**：AI 工具使用 ≥ 2 分，产品思维 ≥ 2 分
- **Stage 2 完成标准**：前端开发 ≥ 2 分，后端开发 ≥ 2 分，部署运维 ≥ 1 分
- **Stage 3 完成标准**：所有维度 ≥ 2 分，至少一个维度达到 3 分

## 学习建议

1. **坚持"完成比完美更重"**：先跑通再优化
2. **多问 AI**：遇到问题第一时间问 AI，养成"遇到问题→描述问题→AI解答"的习惯
3. **做好笔记**：每个知识点完成后记录关键收获
4. **定期回顾**：每周末花 30 分钟回顾本周所学
5. **产出作品**：每个阶段都要有可展示的产出
