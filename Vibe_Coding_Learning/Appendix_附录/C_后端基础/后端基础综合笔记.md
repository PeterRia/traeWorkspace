# 附录 C：后端基础

## 01 后端分层架构

```
客户端 (Browser/App)
    ↕ HTTP/HTTPS
API 层 (路由、参数校验)
    ↕
业务逻辑层 (核心功能)
    ↕
数据访问层 (数据库交互)
    ↕
数据库 (存储)
```

### Vibe Coding 中的后端
- 用自然语言描述 API 需求
- AI 生成路由和数据处理代码
- 需要理解基本的分层概念

## 03 API 设计入门

### RESTful API 基础
```
GET    /api/users      → 获取用户列表
GET    /api/users/1    → 获取用户 #1
POST   /api/users      → 创建新用户
PUT    /api/users/1    → 更新用户 #1
DELETE /api/users/1    → 删除用户 #1
```

### API 响应格式
```json
{
  "status": "success",
  "data": {},
  "message": "操作成功"
}
```
