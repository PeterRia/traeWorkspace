# 01 学习地图 - 理论笔记：Vibe Coding 概述

> 参考：https://datawhalechina.github.io/easy-vibe/zh-cn/stage-1/learning-map/

## 一、什么是 Vibe Coding？

### 定义
**Vibe Coding（氛围编程）**：通过自然语言对话与 AI 协作完成编程项目的方式。核心是"用说话来编程"——你不需要直接写代码，而是依赖与 AI 对话来完成编程任务。

### 与传统编程的对比

| 维度 | 传统编程 | Vibe Coding |
|------|---------|-------------|
| 入门门槛 | 需要学编程语言、语法、框架 | 会用自然语言描述需求即可 |
| 开发方式 | 手动编写每一行代码 | 对话式描述，AI 生成代码 |
| 效率 | 取决于个人编码速度 | AI 辅助，效率提升 55% |
| 错误处理 | 自己排查和修复 | 截图/复制给 AI，让 AI 修复 |
| 适用场景 | 所有开发任务 | 原型/Demo/自用工具为主 |

### 关键数据
- GitHub Copilot：**1500万+** 开发者正在使用 AI 辅助编程
- 平均 **46%** 的代码由 AI 生成（Java 项目达 61%）
- 开发者任务完成速度提升 **55%**（从 9.6 天缩短到 2.4 天）
- 首日安装率达 **81%**，建议采纳率 **96%**

## 二、Vibe Coding 的核心挑战

### 让 AI 写出代码只是第一步，你还会遇到：
1. **代码质量**：怎么让 AI 写出干净、能维护的代码？
2. **系统整合**：怎么把零散的代码拼成一个能跑的应用？
3. **部署上线**：怎么让应用真正上线、被人用到？
4. **AI 集成**：怎么把文本生成、图像识别等 AI 能力装进你的产品？

## 五、项目制学习的价值

### 为什么用项目制？
现实工作场景：
- 需求是模糊的，边界是变化的
- 没有标准答案，需要自己查资料、做实验
- 需要从创意到上线的完整闭环

### 你获得的训练：
- 拆解问题、设计方案、自己寻找资料
- 理解、阅读和改造中大型代码库
- 体验真实产品从 0 到 1 的完整过程

## 六、提问的艺术

### 为什么提问是必备技能？
- 现实很少有完整文档
- AI 是你随身的导师 + 同事
- 你的能力上限由沟通决定

### 信息输入方式对比

| 方式 | 适用场景 | 关键要求 |
|------|---------|---------|
| 复制粘贴 | 报错堆栈、日志、代码、配置 | 尽量完整，不要只截一行 |
| 截图 | UI 布局问题、交互异常 | 截全屏 + 标注重点区域 |

### 学习型提问模板
```
"请先用 5 句话讲清楚这个概念，再给几个问题提问我验证我理解对了没。"
"请你详细解释一下这个报错信息，我不理解为什么会报错。"
```
