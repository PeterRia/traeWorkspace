# 02 游戏化入门 - 练习题

## 基础练习（必做）

### 练习 1：首次 Vibe Coding 体验
1. 在 z.ai 中完成基础版贪吃蛇（60秒版本）
2. 记录：生成用了多长时间？效果满意吗？
3. 如果遇到问题，你是怎么解决的？

### 练习 2：功能迭代
在基础版贪吃蛇上逐步添加功能：
- [ ] 添加分数显示
- [ ] 添加游戏速度递增
- [ ] 添加最高分记录（localStorage）
- [ ] 美化界面

每完成一个功能，记录用了多少轮对话。

### 练习 3：AI 调试实践
故意在游戏中制造以下问题，然后让 AI 修复：
1. 修改一处 CSS 让按钮位置错乱 → 截图给 AI 修复
2. 修改一处 JS 让蛇无法转向 → 复制代码给 AI 修复
3. 修改游戏结束条件 → 描述需求给 AI 添加新规则

## 进阶练习（选做）

### 练习 4：AI 原生贪吃蛇
完成 AI 原生版的贪吃蛇（吃单词→写诗→画画），记录：
- 实现过程中遇到的最大挑战是什么？
- AI 生成的诗和画质量如何？
- 你对 Vibe Coding 有什么新的认识？

### 练习 5：自创 AI 小游戏
选择一个创意，用 Vibe Coding 实现：
- [ ] AI 艺术画廊
- [ ] 复古游戏档案馆
- [ ] 极简任务管理系统
- [ ] 个人知识图谱
- [ ] 其他你自己的想法

### 练习 6：提示词对比实验
用同一个需求，写三种不同质量的提示词给 AI：
1. 模糊版本（如"做个游戏"）
2. 中等版本（列几个功能点）
3. 详细版本（包含具体交互、设计、约束）

对比三种结果的差异，总结写好提示词的关键要素。

## 思考题

1. 在本次实践中，你觉得哪些是 AI 做得很好的？
2. 哪些地方让你觉得 AI 还不够好？
3. 你觉得 Vibe Coding 和"让 AI 替你做"有什么区别？
4. 如果给一个完全不会编程的朋友推荐 Vibe Coding，你会怎么介绍？

---

## 答案要点

### 思考题 1 参考
AI 擅长：生成标准组件、实现常见交互、快速原型、代码解释和修复

### 思考题 2 参考
AI 不足之处：复杂业务逻辑、性能优化、安全设计、产品细节打磨

### 思考题 3 参考
Vibe Coding 不是"AI 替你全做完"，而是你通过对话引导 AI，你负责决策和方向，AI 负责执行和实现。这是一种协作关系而非替代关系。
