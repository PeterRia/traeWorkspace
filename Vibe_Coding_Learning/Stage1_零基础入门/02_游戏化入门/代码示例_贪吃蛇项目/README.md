# 02 游戏化入门 - 贪吃蛇项目实践

> 参考：https://datawhalechina.github.io/easy-vibe/zh-cn/stage-1/ai-capabilities-through-games/

## 项目目标
通过制作 AI 原生贪吃蛇游戏，体验完整的 Vibe Coding 工作流。

## 第一步：60 秒贪吃蛇（基础版）

### 在 z.ai 中使用的提示词
```
帮我做一个贪吃蛇游戏：
1. 用方向键控制蛇的移动
2. 吃到食物后蛇会变长，分数增加
3. 撞到墙壁或自己的身体就游戏结束
4. 要有开始和重新开始按钮
5. 界面要简洁好看
```

### 操作步骤
1. 打开 https://z.ai
2. 在对话框中输入上述提示词
3. 点击"全栈开发"按钮
4. 等待生成完成（约 1-2 分钟）
5. 在右侧预览窗口查看效果
6. 点击指南针图标切换到全屏模式

### z.ai 界面按钮说明
| 按钮 | 功能 |
|------|------|
| 箭头 ↔ | 展开/收起对话历史 |
| 铅笔 ✏️ | 新建对话 |
| 循环箭头 🔄 | 刷新页面 |
| 指南针 🧭 | 全屏预览 |
| Download ⬇ | 下载项目代码 |
| 代码 <> | 查看/切换代码视图 |
| Publish 📤 | 发布项目 |

## 第二步：AI 原生贪吃蛇（升级版）

### 第一阶段提示词：基础玩法
```
帮我做一个贪吃蛇游戏，它应该支持：
1. 蛇可以吃不同的单词，它们会被收集在一个盒子里
```

### 第二阶段提示词：AI 功能集成
```
帮我做一个贪吃蛇游戏，它应该支持：
1. 我可以吃不同的单词，它们会被收集在一个盒子里
2. 当蛇吃了 8 个单词时，大模型应该根据这些单词创作一首诗，我们可以根据需要重新混合这首诗
3. 当诗完成后，下一步将自动根据这首诗创建一幅图像
```

### 操作要点
- 每轮生成后先测试，确认功能正常再添加新需求
- 如果遇到 bug：复制错误信息 + 描述期望效果 → 发给 AI
- 如果 UI 不满意：截图 + 描述修改方向 → 发给 AI

## 第三步：给游戏添加创意功能

### 创意想法列表（选一个实现）

1. **单词解锁世界**：每个单词触发 AI 生成小艺术品，拼成全景图
2. **诗歌拼图**：吃掉的单词组合成一首 AI 协作诗
3. **魔法单词**：特殊单词改变游戏主题和画风
4. **主题关卡**：选择主题（童话/科幻/唐诗），AI 调整风格
5. **故事分支**：玩家的单词选择影响故事走向
6. **创作分享**：保存并分享 AI 创作的诗和画

## AI 调试技巧

### 遇到问题时的提问模板
```
问题描述：[具体描述现象]
当前代码：[粘贴相关代码]
错误信息：[粘贴报错]
期望效果：[描述你希望的效果]
```

### 常见问题与解决
| 问题 | 可能原因 | 解决方向 |
|------|---------|---------|
| 点击按钮没反应 | JS 事件未绑定 | 让 AI 检查事件监听 |
| 功能未按预期工作 | 逻辑错误 | 详细描述预期和实际差异 |
| 页面样式不对 | CSS 问题 | 截图 + 描述期望布局 |
| 诗歌生成失败 | API 调用问题 | 让 AI 添加错误处理 |
