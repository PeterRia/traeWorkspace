# 03 产品原型实战 - 理论笔记：需求挖掘方法论

> 参考：https://datawhalechina.github.io/easy-vibe/zh-cn/stage-1/finding-great-idea/

## 一、为什么先学"找需求"而不是"写代码"？

### 独立开发者小明的故事
小明花一年做了健身 APP，功能齐全、界面精美，但上线后：
- 5 万下载，7 日留存仅 5%
- 几乎没有付费用户
- 亏了 20 万

**失败原因**：方向错了，越深入错得越多。他从来没问过"用户真的需要吗？"

### 核心教训
既然 Vibe Coding 让我们能快速把想法变成产品，我们更应该学会找到值得做的想法。做"有人愿意用的产品"而不是"练习项目"。

## 二、需求的三层模型

### 痛点（Pain Point）—— 恐惧驱动
- 本质：用户正在经历的痛苦、焦虑、不便
- 特点：不解决会很难受，用户不得不买单
- 类型：**止痛药**
- 案例：小餐馆老板凌晨 4 点起床买菜（恐惧：生存艰辛）

### 爽点（Delight Point）—— 即时满足
- 本质：需求能够立即被满足，产生即时愉悦
- 特点：让用户"爽"是留存的关键
- 类型：**海洛因**（褒义的上瘾机制）
- 案例：外卖 30 分钟送达（即时满足饥饿）

### 痒点（Itch Point）—— 虚拟自我
- 本质：想要变得更好，但不是必须的
- 特点：满足了开心，不满足也没事
- 类型：**维生素 / 奢侈品**
- 案例：记录每天喝了多少水（想象中的自律生活）

### 优先级排序
```
痛点 > 爽点 > 痒点
```

## 三、验证真需求的 5 步法

### 第一步：与真实用户聊天
找到 10 个目标用户，问："你现在怎么解决这个问题？"
- 如果用户已经在用某种方法 → 问题真实存在
- 如果用户说不需要 → 可能不是真需求

### 第二步：分析替代方案
用户的替代方案：其他产品 / Excel / 靠记忆 / 忍受
- 你的产品要比替代方案好很多，用户才愿意换

### 第三步：测试付费意愿
做预售或收定金：
- 付费意愿 > 10%：需求真实，值得投入
- 5%-10%：需求存在，需打磨
- < 5%：需求可能不成立

### 第四步：估算市场规模
```
市场规模 = 目标用户总数 × 付费意愿 × 客单价
```

### 第五步：思考竞争壁垒
- 技术难度
- 网络效应
- 品牌
- 成本优势

## 四、从想法到好点子的方法

### 1. 从自身出发
- 你自己就是用户，天然理解痛点
- 你的爱好是最好的起点

### 2. 横向切分人群
不要服务"所有人"，找到"最痛的那个群体"
- 健身人群 → 产后妈妈（更精准、更痛）
- 饮食记录 → 糖尿病患者（更刚需）

### 3. 纵向深挖场景
描述完整的用户旅程：
- 使用前：用户在想什么？在做什么？
- 使用中：遇到什么困难？
- 使用后：得到了什么？还有什么问题？

### 4. 价值重构
- 从"功能"升级为"解决方案"
- 从"工具"升级为"助手/管家/伙伴"

## 五、产品经理的场景思维

### 真正的痛点 = 场景中的情绪 + 付费意愿
- 恐惧 → 担心身体恢复不好，留下后遗症
- 焦虑 → 看着镜子里的自己，觉得人生完蛋了
- 无助 → 不知道从何开始，没人指导
- 孤独 → 别人随便生，我却要恢复这么久

**好的产品解决的是情绪，而不只是功能。**

## 六、需求验证的常见误区

| 误区 | 正确做法 |
|------|---------|
| "我觉得有用" | 用户觉得有用才行 |
| "市场很大" | 市场大 ≠ 你能分到 |
| "功能越多越好" | 聚焦核心痛点 |
| "免费就会有人用" | 免费也未必有人用 |
| 用痛点方式推销痒点产品 | 痛点产品用痛点方式，痒点产品用痒点方式 |

## 关键概念速查

| 术语 | 含义 |
|------|------|
| MVP | 最小可行产品，验证核心假设的最小版本 |
| PMF | 产品市场契合度（Product-Market Fit） |
| 留存率 | 用户持续使用产品的比例 |
| 客单价 | 每个用户平均消费金额 |
| ARPU | 每用户平均收入（Average Revenue Per User） |
