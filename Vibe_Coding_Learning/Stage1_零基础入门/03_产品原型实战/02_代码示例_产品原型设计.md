# 03 产品原型实战 - 代码示例：产品原型设计

## 示例一：用 AI 打磨产品概念

### 场景
你有了一个产品想法（如"产后妈妈恢复助手"），需要 AI 帮你完善。

### 第一步：描述初始想法
```
我有一个产品想法：做一个帮助产后妈妈恢复身材的 APP。
请帮我分析：
1. 这个想法解决的核心问题是什么？
2. 目标用户是谁？他们的典型特征是什么？
3. 市面上已有的解决方案有哪些？有什么不足？
4. 这个产品的差异化优势应该是什么？
```

### 第二步：让 AI 设计产品功能
```
基于以下信息，帮我设计产品功能：
- 目标用户：产后 0-12 个月的妈妈
- 核心痛点：身体恢复焦虑、时间碎片化、缺乏专业指导
- 差异化：AI 动作纠正、产后专属课程、心理支持

请给出：
1. 核心功能列表（MVP 版本）
2. 每个功能的详细描述
3. 功能优先级排序（P0/P1/P2）
4. 建议的技术实现方案
```

### 第三步：让 AI 评估商业可行性
```
基于这个产品概念，请分析：
1. 市场规模估算（中国每年新生儿约 1000 万）
2. 推荐的定价策略
3. 潜在的竞争壁垒
4. 获客渠道建议
5. 风险点和应对策略
```

## 示例二：生成需求文档（PRD）框架

### 提示词
```
请根据以下信息，生成一份简单的产品需求文档（PRD）：

产品名称：产后妈妈恢复助手
目标用户：产后 0-12 个月的妈妈
核心痛点：
- 没时间健身（需要照顾宝宝）
- 身体有限制（腹直肌分离等）
- 缺乏专业指导
- 心理焦虑

请包含：
1. 产品概述
2. 目标用户画像
3. 核心功能列表
4. 用户使用流程
5. 关键页面设计要点
6. 技术需求
```

## 示例三：快速原型生成

### 提示词
```
帮我生成一个产后妈妈恢复助手的产品落地页：

页面需要包含：
1. 标题和副标题（突出核心价值）
2. 3 个核心功能展示模块
3. 用户评价/案例展示区
4. 价格方案（免费/基础/高级）
5. 注册/下载入口
6. 常见问题 FAQ

设计要求：
- 温暖柔和的色调（粉色系 + 米白色）
- 圆润的边角设计
- 适合妈妈群体的简洁排版
- 移动端优先的响应式设计
```

## 关键提示词模式

### 需求分析模式
```
分析我的产品想法 [产品名]：
- 目标用户是 [用户描述]
- 核心问题是 [问题描述]
- 请从需求验证、市场规模、竞争分析三个维度给出建议
```

### 概念验证模式
```
我想验证一个产品概念：[一句话描述]
请帮我设计一个验证方案，包括：
1. 如何找到前 10 个用户
2. 如何设计访谈问题
3. 如何判断需求是否真实
4. 什么信号说明应该放弃
```
