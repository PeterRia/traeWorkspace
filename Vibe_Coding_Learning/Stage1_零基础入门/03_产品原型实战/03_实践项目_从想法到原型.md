# 03 产品原型实战 - 实践项目：从想法到原型

## 项目目标
运用需求挖掘方法论 + Vibe Coding，独立完成一个产品从创意到原型的完整闭环。

## 项目流程

### Phase 1：找方向（1 天）
#### 任务清单
- [ ] 列出你生活中遇到的 10 个"不爽"的点
- [ ] 用 5 步验证法筛选出 3 个值得做的方向
- [ ] 对每个方向进行横向切分（找精准人群）
- [ ] 从 3 个中选出 1 个最值得做的

#### 选方向检查
- [ ] 你自己或身边人是否深受其害？（自我验证）
- [ ] 是否已经有了替代方案？（需求真实性）
- [ ] 目标人群是否足够精准？（纵向深挖）
- [ ] 你是否有独特的理解或优势？（竞争壁垒）

### Phase 2：需求验证（1 天）
#### 用户访谈清单
- [ ] 找到 5-10 个目标用户
- [ ] 了解他们"现在怎么解决这个问题"
- [ ] 了解替代方案的缺点
- [ ] 试探付费意愿（"如果收费 X 元/月，你会用吗？"）

#### 验证结果判断
```
访谈结果：____ / 10 人表示需要
付费意愿：____ / 10 人表示愿意付费
结论：[继续 / 调整方向 / 放弃]
```

### Phase 3：产品设计（1 天）
#### 用 AI 完善产品概念
- [ ] 用 AI 分析竞品和差异化
- [ ] 用 AI 设计 MVP 功能列表
- [ ] 用 AI 评估商业模式

#### 核心文档产出
- [ ] 一句话产品定位
- [ ] 目标用户画像
- [ ] MVP 功能列表（P0/P1/P2）
- [ ] 关键页面线框图（文字描述即可）

### Phase 4：原型开发（2-3 天）
#### 用 Vibe Coding 开发原型
- [ ] 生成产品落地页（介绍核心功能）
- [ ] 生成核心功能页面
- [ ] 添加基本交互逻辑
- [ ] 美化界面（用时下流行设计风格）

#### 原型质量标准
- [ ] 可以打开并能浏览全部页面
- [ ] 核心功能可以交互
- [ ] 界面美观，专业感强
- [ ] 有人对原型表示"想要用"
