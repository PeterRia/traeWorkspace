# 05 部署上线 - 理论笔记

## 部署平台对比

| 平台 | 特点 | 适用场景 | 免费额度 |
|------|------|---------|:---:|
| Vercel | 前端为主，自动部署 | React/Next.js 项目 | ✅ |
| Zeabur | 全栈部署，简单易用 | 完整 Web 应用 | ✅ |
| CloudBase | 腾讯云出品，国内友好 | 国内项目 | ✅ |
| Netlify | 静态网站部署 | 纯前端项目 | ✅ |
| Railway | 全栈部署 | 数据库+后端 | 有限 |

## 部署流程核心步骤

1. **代码准备**：确保项目在本地正常运行
2. **推送代码**：将项目推送到 GitHub
3. **连接平台**：在部署平台关联 GitHub 仓库
4. **配置环境**：设置环境变量、数据库连接等
5. **一键部署**：平台自动构建和部署
6. **域名配置**：绑定自定义域名（可选）

## 关键概念

- **CI/CD**：持续集成/持续部署，代码推送自动触发部署
- **环境变量**：存储敏感信息（API Key、数据库密码等）
- **构建命令**：部署平台执行的项目构建指令
