# 一键交给 Codex 的启动提示词

> 使用方式：把下面“提示词正文”整段复制给 Codex。建议在一个空目录中启动 Codex，让它直接生成完整项目。

---

## 提示词正文

你是 Codex，请在当前空目录中一次性创建并完成一个 Python 拼图游戏项目。请不要只给思路，必须直接创建可运行源码、资源、测试、说明文档和 Windows exe 打包方案。项目最终用于课程作业，主题是“使用 Pillow、pygame、random、os.path、datetime 等库设计一款滑动拼图游戏”。

### 一、总目标

创建一个完整项目文件夹 `puzzle_game/`，实现一款好玩的滑动拼图游戏：玩家选择图片和难度后进入游戏，通过点击或拖拽与空白块相邻的拼块完成拼图；游戏实时显示原图参考、用时、步数、提示次数、当前模式；完成后显示结算页、保存成绩、解锁新图片，并能重新开局。最终请生成可运行的 `main.py`、资源文件、README、用户手册、测试代码、打包脚本，并尽力用 PyInstaller 生成 exe。

### 二、技术栈与硬性约束

1. 必须使用 Python 3.10+。
2. 必须使用 `pygame` 负责窗口、事件、绘制、音效。
3. 必须使用 `Pillow/PIL` 负责原图生成、裁剪、缩放、成绩卡片生成。
4. 必须使用 `random` 完成随机打乱，但打乱必须保证可解。
5. 必须使用 `os.path` 处理资源路径和保存路径，兼容源码运行与 PyInstaller exe 运行。
6. 必须使用 `datetime` 记录成绩日期、生成成绩卡片水印和日志时间。
7. 不依赖网络，不使用有版权风险的外部图片或音频。若缺少素材，请用 PIL 和标准库 `wave/math` 自动生成 6 张主题图片和若干简单 wav 音效。
8. UI 文案使用中文；字体优先使用系统中文字体，找不到时自动回退。
9. 程序不能因为音频设备、字体缺失、存档缺失、资源缺失而崩溃，必须有降级处理。
10. 不要把存档写入 PyInstaller 的 `_MEIPASS` 临时目录。源码运行时写入项目内 `saves/`，exe 运行时写入 exe 同级 `saves/`。

### 三、功能范围分级

请按下面优先级实现。A 级必须全部完成；B 级尽量完成且不要影响 A 级稳定性；C 级可做简化实现，但必须预留清晰代码结构和文档说明。

#### A 级：必须稳定完成

1. 游戏窗口与状态机：主菜单、图片/难度选择、游戏中、暂停、胜利结算、失败/返回。
2. 图像处理：加载或生成 6 张主题图片；将图片缩放裁剪为正方形；按 3x3、4x4、5x5 切成拼块；最后一格为空白块。
3. 可解打乱：不能直接 `random.shuffle`。必须从完成状态出发随机执行合法移动若干步，确保每局必有解。
4. 点击操作：只响应拼图区域内点击；点击与空白块上下左右相邻的拼块才移动；点击其他区域或不可移动拼块不改变棋盘。
5. 完成判断：所有拼块回到目标位置，且旋转模式下角度也正确，才判定胜利。
6. 原图参考：右侧显示完整原图缩略图，支持显示/隐藏。
7. 计时、计步：实时显示用时和成功移动次数。
8. 重新开局：当前窗口内可重新开始、返回菜单、换图、换难度。
9. 本地存档：使用 JSON 保存设置、图鉴解锁、每图完成次数、最佳时间、最少步数、成就列表。
10. 打包交付：提供 `requirements.txt`、`run_game.bat`、`build_exe.bat`、`README.md`、`用户手册.md`。

#### B 级：增强好玩度

1. 图鉴冒险：初始只解锁第 1 张，完成一张解锁下一张；未解锁图片显示问号或灰色卡片。
2. 成就系统：至少实现“初次尝试”“步数大师”“闪电侠”“完美主义者”“旋转勇者”“不屈不挠”。
3. 提示系统：按 H 或按钮高亮可移动拼块；按 G 给出一个基于曼哈顿距离的下一步建议。提示不直接替玩家完成。
4. 撤销/重做：Ctrl+Z 撤销，Ctrl+Y 重做。撤销次数单独记录，不允许通过撤销刷低步数。
5. 音效反馈：移动、错误点击、提示、胜利、解锁使用不同音效；音频失败时自动静音继续运行。
6. 胜利结算：显示难度、用时、步数、提示次数、撤销次数、评级 S/A/B/C、是否刷新纪录。
7. 成绩卡片：完成后用 PIL 生成 640x360 的 PNG 成绩卡，保存到 `screenshots/`。

#### C 级：可简化但尽量完成

1. 拖拽操作：鼠标拖拽相邻拼块到空白格释放后移动；如实现成本过高，可保留点击模式并在设置中禁用拖拽。
2. 旋转模式：4x4 及以上可开启。拼块可随机旋转；右键或 R 键旋转 90 度；胜利需位置与角度均正确。
3. 限时挑战与限步挑战：可以实现为模式开关；3x3=60秒/20步，4x4=120秒/40步，5x5=180秒/80步。
4. 粒子特效和动态背景：用轻量粒子、闪光、胜利烟花即可，不要为了特效牺牲稳定性。

### 五、核心技术设计要求

1. 棋盘模型必须和 UI 绘制解耦：`src/board.py` 只处理格子、移动、旋转、打乱、胜利判断，不依赖 pygame。
2. 拼块数据建议包含：`tile_id`、`target_row`、`target_col`、`row`、`col`、`rotation`。
3. 空白块用 `None` 表示。
4. `Board.shuffle()` 必须从完成状态开始，随机选择合法相邻块移动。移动次数建议：3x3=80、4x4=200、5x5=400。避免最后仍是完成态。
5. 完成判断：每个非空拼块 `row == target_row and col == target_col`，旋转模式还要 `rotation % 360 == 0`。
6. 曼哈顿提示：计算每个可移动拼块移动前后的总曼哈顿距离，选择能降低最多的拼块；如果没有降低项，则选择降低最少损失或随机合法项。
7. 路径处理必须包含类似逻辑：源码运行时以项目根目录为基础，exe 运行时以 `sys.executable` 所在目录为可写目录，读取 bundled assets 时兼容 `sys._MEIPASS`。
8. 保存文件为 `saves/save_data.json`，程序启动时不存在就创建默认数据；JSON 损坏时备份为 `.bak` 并重建默认数据。
9. 成绩卡保存在 `screenshots/score_YYYYMMDD_HHMMSS.png`。
10. 排行榜按 `difficulty + mode + image_id` 维度保存 Top 10。

### 六、界面与交互要求

1. 默认窗口建议 1100x720，棋盘区域约 600x600，右侧信息栏显示原图、按钮、统计。
2. 主菜单按钮：开始游戏、图鉴、排行榜、设置、退出。
3. 游戏界面按钮：提示 H、最佳提示 G、撤销、重做、暂停、重开、原图显示/隐藏、返回菜单。
4. 快捷键：ESC 暂停，H 高亮，G 最佳提示，Ctrl+Z 撤销，Ctrl+Y 重做，R 旋转当前悬停/选中拼块，M 静音。
5. 所有按钮需要有 hover 状态，点击后有视觉反馈。
6. 非法点击不可移动拼块时，不移动棋盘，可播放轻微错误音效或显示短暂提示。
7. 胜利后弹出结算面板，不要立即关闭窗口。

### 七、评分与评级建议

评级用于增强反馈，不要求非常严格：

```text
score_steps = move_count + hint_count * 5 + undo_count
3x3: S <= 25步且 <= 45秒；A <= 45步且 <= 90秒；B <= 80步；否则 C
4x4: S <= 80步且 <= 150秒；A <= 140步且 <= 240秒；B <= 220步；否则 C
5x5: S <= 180步且 <= 300秒；A <= 300步且 <= 480秒；B <= 480步；否则 C
旋转模式评级可整体放宽 40%。
```

### 八、测试与校验要求

请创建并运行以下检查：

1. `python -m py_compile main.py src/*.py tools/*.py`
2. `python -m unittest discover -s tests`
3. `python tools/generate_assets.py` 能重复执行且不会破坏已有资源。
4. `python tools/smoke_test.py` 使用 `SDL_VIDEODRIVER=dummy` 尝试初始化游戏主要模块，不要求人工交互。
5. `python tools/package_check.py` 检查关键文件存在：源码、assets、README、用户手册、bat 脚本、requirements。
6. 手工测试至少覆盖：3x3 完成、非法点击无响应、重开、返回菜单、存档创建、成绩卡生成、exe 从 dist 目录启动。

### 九、打包要求

1. `requirements.txt` 至少包含：`pygame`、`Pillow`、`pyinstaller`。
2. `build_exe.bat` 使用 PyInstaller 打包，建议命令类似：

```bat
pyinstaller --noconfirm --windowed --name PuzzleGame --add-data "assets;assets" main.py
```

3. 打包后检查 `dist/PuzzleGame/PuzzleGame.exe` 或 `dist/PuzzleGame.exe` 是否能启动。
4. 最终交付说明中写明：把整个 `puzzle_game/` 文件夹和打包好的 exe 一起压缩为 `学号姓名.rar`，例如 `25331021张三.rar`。

### 十、文档交付要求

请在项目内写好：

1. `README.md`：项目介绍、功能列表、环境安装、运行命令、打包命令、目录结构、常见问题。
2. `用户手册.md`：玩家如何开始、难度说明、快捷键、图鉴/成就/排行榜说明、故障排查。
3. 代码内关键函数必须有简明中文注释，尤其是可解打乱、胜利判断、路径处理、存档读写、PyInstaller 兼容。
