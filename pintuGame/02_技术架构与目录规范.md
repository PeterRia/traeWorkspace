# 技术架构与目录规范

## 1. 架构原则

项目采用“模型层 + 资源层 + 界面层 + 持久化层”的轻量架构，避免所有逻辑堆在一个文件里。

- **模型层**：只处理拼图规则，不关心 pygame 绘制。
- **资源层**：负责图片、音效、字体、路径兼容。
- **界面层**：负责按钮、菜单、绘制、事件分发。
- **持久化层**：负责 JSON 存档、排行榜、设置。
- **工具层**：负责素材生成、冒烟测试、交付检查。

## 3. 模块职责

| `ui.py` | 按钮、面板、菜单组件 |

## 4. 状态机设计

建议使用字符串常量或 Enum 管理状态：

```text
MENU           主菜单
SELECT_IMAGE   选择图鉴图片
SELECT_MODE    选择难度和模式
PLAYING        游戏中
PAUSED         暂停
VICTORY        胜利结算
GAME_OVER      挑战失败
COLLECTION     图鉴
LEADERBOARD    排行榜
SETTINGS       设置
```

事件处理只根据当前状态分发，避免“菜单按钮在游戏中误触”等问题。

## 5. 棋盘数据结构

推荐定义：

```python
@dataclass
class Tile:
    tile_id: int
    target_row: int
    target_col: int
    row: int
    col: int
    rotation: int = 0
```

棋盘内部保存：

```python
grid: list[list[int | None]]
tiles: dict[int, Tile]
blank_pos: tuple[int, int]
```

这样既能快速判断某个格子是什么，也能根据 tile_id 找到目标位置。

## 6. 移动规则

- `get_movable_tiles()` 返回与空白块曼哈顿距离为 1 的拼块。
- `can_move(row, col)` 判断目标格是否存在且相邻空白。
- `move(row, col)` 成功时交换 tile 与 blank，返回 True；失败返回 False。
- UI 只有在 `move()` 返回 True 后才增加步数和播放移动音效。

## 7. 可解打乱算法

必须采用“从完成状态随机走合法步”的方式：

```python
for _ in range(shuffle_steps):
    movable = board.get_movable_tiles()
    tile = random.choice(movable)
    board.move_tile(tile.row, tile.col, record=False)
```

优化点：避免刚移动过来的拼块立刻反向移动，可记录上一次空白位置，优先选择非回退动作。

## 8. 旋转模式

- 打乱位置后，再随机旋转若干拼块。
- 空白块无旋转。
- 旋转只影响拼块渲染和胜利判断，不影响移动合法性。
- 旋转操作写入 undo 栈。

## 9. 路径处理规范

需要区分“读取资源路径”和“写入用户数据路径”：

```python
# 读取 assets 时兼容 PyInstaller
def resource_path(relative_path):
    base = getattr(sys, '_MEIPASS', os.path.abspath('.'))
    return os.path.join(base, relative_path)

# 写入 saves/screenshots 时使用可写目录
def runtime_dir():
    if getattr(sys, 'frozen', False):
        return os.path.dirname(sys.executable)
    return os.path.abspath(os.path.dirname(os.path.dirname(__file__)))
```

不要把 JSON 存档写进 `_MEIPASS`，因为那是 PyInstaller 运行时临时目录。

## 10. 存档格式

```json
{
  "version": "1.0",
  "settings": {
    "difficulty": "3x3",
    "control_mode": "click",
    "sound_enabled": true,
    "show_preview": true,
    "rotation_mode": false
  },
  "collection": {
    "unlocked": [0],
    "completed_count": {"0": 0}
  },
  "records": {},
  "leaderboard": {},
  "achievements": [],
  "last_played_at": null
}
```

建议按 `image_id + difficulty + mode` 组合保存成绩，例如：`image_01|3x3|normal`。

## 11. 成绩与评级

- `move_count`：成功移动和旋转次数。
- `hint_count`：提示次数。
- `undo_count`：撤销次数。
- `score_steps = move_count + hint_count * 5 + undo_count`。
- 排行榜同时保存原始步数和 score_steps。

## 12. 资源生成策略

为避免素材缺失：

- `tools/generate_assets.py` 生成 6 张 800x800 主题图。
- 用 PIL 绘制风景、动物、建筑、抽象、科幻、名画风图案。
- 用标准库 `wave` 生成 move/hint/victory/unlock 等 wav。
- 生成脚本可重复执行，已有文件默认不覆盖，除非传入 `--force`。

## 13. 性能与稳定性

- 拼块 Surface 缓存，难度或图片改变时才重建。
- 主循环固定 60 FPS。
- 粒子数量限制，例如最多 150 个。
- 所有文件读写使用 try/except，异常时显示友好提示并继续。
- 测试棋盘逻辑时不启动 pygame 窗口。
