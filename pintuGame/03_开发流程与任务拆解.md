# 开发流程与任务拆解

## 1. 一次性交给 Codex 的执行策略

为了提高一次性成功率，Codex 应按“先核心、后体验、再打包”的顺序工作。不要先写复杂特效，也不要等所有代码完成后才测试。

推荐顺序：

1. 创建目录结构与基础文件。
2. 实现路径处理、配置、素材生成。
3. 实现纯逻辑棋盘模型并写单元测试。
4. 实现图片切块与缓存。
5. 实现 pygame 主循环、菜单和游戏界面。
6. 接入点击移动、计时、计步、胜利判断。
7. 接入存档、图鉴、排行榜。
8. 接入提示、撤销、音效、成绩卡。
9. 接入旋转、挑战模式、粒子等增强项。
10. 编写 README、用户手册、打包脚本。
11. 运行自动化测试和交付检查。
12. 打包 exe 并记录结果。

## 2. 里程碑拆解

| 阶段 | 产物 | 完成标准 |
|---|---|---|
| M1 项目骨架 | 目录、requirements、入口文件 | `python main.py` 能启动空窗口 |
| M2 棋盘模型 | `board.py` + tests | 可移动、打乱、胜利判断测试通过 |
| M3 资源系统 | 默认图片和音效 | 无外部素材也能运行 |
| M4 游戏主流程 | 菜单、选图、游玩、结算 | 3x3 可完整通关 |
| M5 好玩增强 | 图鉴、提示、撤销、音效、评级 | 操作有反馈，数据能保存 |
| M6 挑战亮点 | 旋转/限时/限步/粒子 | 至少完成部分 C 级功能 |
| M7 交付 | exe、README、用户手册、检查表 | dist 中 exe 能启动 |

## 3. Codex 降级策略

当实现复杂功能遇到冲突时，按以下规则降级：

- 拖拽不稳定：保留点击模式，设置中显示“拖拽模式暂未启用”。
- 音频初始化失败：自动静音，不影响游戏。
- 旋转模式 bug 较多：只在 4x4/5x5 开启，默认关闭。
- 粒子影响性能：减少粒子数量或只保留胜利烟花。
- PyInstaller 打包失败：保留完整源码运行方式，输出失败原因和修复命令。

## 4. 每阶段自检

### M2 棋盘模型自检

- 完成状态下 `is_solved()` 为 True。
- 打乱后 `is_solved()` 通常为 False。
- 非相邻拼块无法移动。
- 相邻拼块移动后空白格位置更新。
- 随机打乱不破坏 tile_id 与目标坐标。

### M4 游戏主流程自检

- 开始游戏后显示拼图区域和右侧原图。
- 点击空白块相邻拼块可移动。
- 点击不可移动拼块无响应。
- 步数只在有效移动后增加。
- 完成拼图后进入结算页。

### M7 打包自检

- `python main.py` 可启动。
- `run_game.bat` 可启动。
- `build_exe.bat` 能执行。
- exe 从 dist 目录启动后能找到 assets。
- exe 启动后能创建 saves 和 screenshots。

## 5. 建议的开发命令

```bash
python -m pip install -r requirements.txt
python tools/generate_assets.py
python -m py_compile main.py src/*.py tools/*.py
python -m unittest discover -s tests
python tools/smoke_test.py
python tools/package_check.py
python main.py
```

Windows 打包：

```bat
build_exe.bat
```

## 6. 代码质量要求

- 关键函数必须短小清晰，避免一个 `game.py` 几千行。
- 纯逻辑函数尽量可测试。
- 所有文件路径通过 `paths.py` 处理。
- 所有 JSON 读写要有异常保护。
- UI 文案统一中文，不出现调试用英文占位。
- 不要硬编码用户电脑绝对路径。

## 7. 收尾报告要求

Codex 完成后必须输出：

1. 项目目录树。
2. 功能完成情况。
3. 测试命令与结果。
4. exe 路径。
5. 已知问题和降级点。
6. 最终压缩为 `学号姓名.rar` 的步骤。
