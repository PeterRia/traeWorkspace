# 校验、验收与测试清单

## 1. 自动化检查清单

| 编号 | 命令 | 通过标准 |
|---|---|---|
| Q1 | `python -m py_compile main.py src/*.py tools/*.py` | 无语法错误 |
| Q2 | `python -m unittest discover -s tests` | 单元测试全部通过 |
| Q3 | `python tools/generate_assets.py` | assets/images 和 assets/sounds 生成成功 |
| Q4 | `python tools/smoke_test.py` | 无窗口或 dummy 窗口下初始化通过 |
| Q5 | `python tools/package_check.py` | 关键文件齐全 |
| Q6 | `python main.py` | 游戏可人工启动 |
| Q7 | `build_exe.bat` | dist 中生成 exe |

## 2. 棋盘逻辑验收

- [ ] 3x3、4x4、5x5 都能生成棋盘。
- [ ] 每局只有一个空白块。
- [ ] 空白块不绘制图片。
- [ ] 拼块数量为 `rows * cols - 1`。
- [ ] 随机打乱后棋盘不是完成态。
- [ ] 打乱方法从完成态执行合法移动，保证可解。
- [ ] 不相邻拼块不能移动。
- [ ] 相邻拼块移动后位置正确交换。
- [ ] 胜利判断准确。
- [ ] 旋转模式下角度未归零不能胜利。

## 3. 用户交互验收

- [ ] 点击拼图区域外不响应。
- [ ] 点击右侧按钮不会误触拼块。
- [ ] 点击空白块本身不增加步数。
- [ ] 点击不可移动拼块不增加步数。
- [ ] 成功移动后步数 +1。
- [ ] 计时器在游戏开始后递增。
- [ ] 暂停时计时停止或不计入结算。
- [ ] ESC 可暂停/返回。
- [ ] H 高亮可移动拼块。
- [ ] G 显示下一步建议。
- [ ] Ctrl+Z/Ctrl+Y 可撤销/重做。
- [ ] 重新开局不会关闭窗口。

## 4. 界面验收

- [ ] 主菜单按钮清晰可点击。
- [ ] 图片选择页能区分已解锁/未解锁。
- [ ] 难度选择能切换 3x3/4x4/5x5。
- [ ] 右侧原图参考清晰。
- [ ] 当前用时、步数、提示次数显示正确。
- [ ] 胜利结算页显示完整统计。
- [ ] 排行榜或最佳记录能查看。
- [ ] 字体不乱码。
- [ ] 窗口关闭按钮可正常退出。

## 5. 存档验收

- [ ] 首次启动自动创建 `saves/save_data.json`。
- [ ] 完成拼图后保存完成次数。
- [ ] 刷新最佳时间时更新存档。
- [ ] 刷新最少步数时更新存档。
- [ ] 解锁下一张图后重启仍保持解锁。
- [ ] 音效开关、操作模式、难度偏好重启后保留。
- [ ] JSON 损坏时能备份并重建，不崩溃。

## 6. 音效与资源验收

- [ ] 无 assets 时可通过生成脚本重建。
- [ ] 图片切块比例正确，不拉伸变形。
- [ ] 音效文件缺失时不崩溃。
- [ ] 没有音频设备时游戏仍可玩。
- [ ] exe 中能读取 bundled assets。

## 7. 成绩卡验收

- [ ] 胜利后在 `screenshots/` 生成 PNG。
- [ ] 文件名含日期时间。
- [ ] 卡片包含原图缩略图、难度、用时、步数、评级、日期。
- [ ] 连续完成多局不会覆盖旧成绩卡。

## 8. 打包验收

- [ ] `requirements.txt` 完整。
- [ ] `run_game.bat` 可运行源码版。
- [ ] `build_exe.bat` 可调用 PyInstaller。
- [ ] dist 内 exe 可打开主菜单。
- [ ] dist 内 exe 能开始游戏。
- [ ] dist 内 exe 能保存记录和成绩卡。
- [ ] 最终压缩包内包含项目文件夹和 exe。

## 9. 人工通关验收建议

为了快速验证通关流程，建议 Codex 在开发时保留一个调试快捷键，例如 F12 直接恢复完成状态并触发胜利。最终提交前可以保留但需在 README 中说明“调试键仅用于验收测试”，或直接禁用。

人工测试流程：

1. 启动游戏。
2. 进入 3x3 普通模式。
3. 移动 3-5 步，确认步数与计时变化。
4. 使用 H/G 提示各一次。
5. 使用 Ctrl+Z 撤销一次。
6. 完成拼图或使用调试流程触发胜利。
7. 检查结算页、存档、成绩卡。
8. 重启游戏，检查记录仍在。
9. 打开 exe 重复第 1-7 步。

## 10. 最终合格标准

- A 级功能全部通过。
- B 级功能至少 5 项可用。
- C 级功能至少 1 项可用或有合理降级说明。
- 源码版和 exe 版至少有一种可稳定启动；课程提交应优先保证 exe 可启动。
- 文档能让老师或同学在 3 分钟内知道如何运行。
