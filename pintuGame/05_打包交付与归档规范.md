# 打包交付与归档规范

## 1. 最终提交目标

课程要求提交压缩包，文件名为：

```text
学号姓名.rar
```

示例：

```text
25331021张三.rar
```

压缩包内建议包含完整项目文件夹和已经打包好的 exe。不要只交 exe，也不要只交源码。

## 2. 推荐最终目录

```text
25331021张三.rar
└── puzzle_game/
    ├── main.py
    ├── requirements.txt
    ├── README.md
    ├── 用户手册.md
    ├── run_game.bat
    ├── build_exe.bat
    ├── assets/
    ├── src/
    ├── tests/
    ├── tools/
    ├── saves/
    ├── screenshots/
    └── dist/
        └── PuzzleGame/
            └── PuzzleGame.exe
```

若 PyInstaller 生成的是单文件模式，也可以是：

```text
puzzle_game/dist/PuzzleGame.exe
```

## 3. requirements.txt

建议至少包含：

```text
pygame>=2.5
Pillow>=10.0
pyinstaller>=6.0
```

## 4. 源码运行说明

Windows：

```bat
python -m pip install -r requirements.txt
python tools\generate_assets.py
python main.py
```

或直接双击：

```text
run_game.bat
```

## 5. exe 打包说明

推荐 `build_exe.bat`：

```bat
@echo off
chcp 65001 >nul
echo 正在安装依赖...
python -m pip install -r requirements.txt

echo 正在生成默认素材...
python tools\generate_assets.py

echo 正在打包 exe...
pyinstaller --noconfirm --windowed --name PuzzleGame --add-data "assets;assets" main.py

echo 打包完成，请检查 dist\PuzzleGame\PuzzleGame.exe
pause
```

如果采用单文件打包：

```bat
pyinstaller --noconfirm --onefile --windowed --name PuzzleGame --add-data "assets;assets" main.py
```

单文件启动略慢，但提交更简洁；目录模式更稳，资源读取问题更少。

## 6. RAR 压缩步骤

### 方式 A：WinRAR 图形界面

1. 右键 `puzzle_game/` 文件夹。
2. 选择“添加到压缩文件”。
3. 压缩文件名填 `学号姓名.rar`。
4. 确认格式为 RAR。
5. 点击确定。

### 方式 B：命令行

如果电脑已安装 WinRAR，并且 `rar.exe` 在 PATH 中：

```bat
rar a 25331021张三.rar puzzle_game
```

或使用 WinRAR 安装目录：

```bat
"C:\Program Files\WinRARar.exe" a 25331021张三.rar puzzle_game
```

## 7. 提交前删除或保留

建议保留：

- `assets/`
- `src/`
- `tests/`
- `tools/`
- `README.md`
- `用户手册.md`
- `dist/` 中的 exe
- 空的 `saves/` 和 `screenshots/` 目录

可以删除：

- `__pycache__/`
- `.pytest_cache/`
- `build/`
- 过多测试截图
- 临时日志

是否保留 `.spec`：建议保留 PyInstaller 生成的 `PuzzleGame.spec`，方便老师复现打包。

## 8. 交付前最终检查

- [ ] 压缩包文件名是 `学号姓名.rar`。
- [ ] 解压后能看到 `puzzle_game/` 文件夹。
- [ ] `puzzle_game/main.py` 存在。
- [ ] `puzzle_game/dist/` 内有 exe。
- [ ] 双击 exe 能打开游戏。
- [ ] README 写明运行方式。
- [ ] 用户手册写明玩法。
- [ ] 没有绝对路径依赖。
- [ ] 没有引用本机私有文件。

## 9. 常见问题

### 9.1 exe 打开后闪退

可能原因：资源路径错误、缺少 assets、pygame 初始化失败。解决：

- 从命令行启动 exe 查看报错。
- 检查 `paths.py` 是否兼容 `_MEIPASS`。
- 确认 PyInstaller 命令包含 `--add-data "assets;assets"`。

### 9.2 中文乱码

- 使用 `pygame.font.SysFont` 时提供中文字体列表。
- README 中说明 Windows 推荐安装微软雅黑或黑体。

### 9.3 音效报错

- mixer 初始化失败时自动禁用音效。
- 不要让音效错误中断游戏。

### 9.4 存档无法写入

- exe 版不要写 `_MEIPASS`。
- 使用 exe 同级目录作为 `saves/` 写入位置。
