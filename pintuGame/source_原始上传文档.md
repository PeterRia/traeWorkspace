# 拼图游戏扩充功能设计文档

## 一、设计哲学

本扩充方案以「**让拼图从静态作业变成可重复游玩的休闲游戏**」为核心目标，引入**进度感、操作爽感、视觉反馈、挑战梯度**四大设计维度，将传统拼图升级为具备完整游戏循环的轻量级独立游戏。

---

## 二、扩充功能清单

### 2.1 进度与叙事系统（解决"玩完就丢"问题）

#### 图鉴冒险模式（Collection Mode）
- **机制**：内置6张不同主题的原图（风景、名画、动物、建筑、抽象艺术、科幻），初始仅解锁第1张。
- **解锁规则**：完成当前拼图后自动解锁下一张，进度自动保存到本地JSON。
- **图鉴界面**：主菜单增设「图鉴」按钮，可浏览已解锁/未解锁图片，未解锁图片显示为灰色剪影+问号。
- **完成度**：每张图记录最佳时间、最少步数、完成次数。

#### 成就系统（Achievement System）
| 成就名称 | 触发条件 | 奖励/反馈 |
|---------|---------|----------|
| 初次尝试 | 完成任意一张拼图 | 解锁图鉴第2张 |
| 步数大师 | 3×3难度下20步内完成 | 称号显示 |
| 闪电侠 | 3×3难度下30秒内完成 | 金色边框特效 |
| 完美主义者 | 完成全部6张图鉴 | 隐藏第7张彩蛋图解锁 |
| 旋转勇者 | 在旋转模式下完成任意拼图 | 特殊粒子颜色 |
| 不屈不挠 | 连续撤销超过10次后仍完成 | 鼓励性提示 |

---

### 2.2 操作体验升级（解决"操作生硬"问题）

#### 双操作模式（Dual Control Scheme）
- **点击模式（Click Mode）**：点击与空白块相邻的拼块，自动交换位置。适合传统玩家。
- **拖拽模式（Drag Mode）**：按住拼块直接拖拽到空白位置（或相邻位置），释放后交换。符合现代触屏/鼠标直觉。
- **设置持久化**：玩家偏好的操作模式保存到本地，下次启动自动加载。

#### 拼块旋转模式（Rotation Hell Mode）
- **机制**：在4×4及以上难度可选开启。拼块除了位置错乱，本身还可能被旋转90°/180°/270°。
- **操作**：右键单击（或按R键）旋转当前拼块90°。只有位置和角度都正确才算归位。
- **难度系数**：开启后拼块状态空间从 n! 扩展到 n! × 4^n，极大提升挑战性。

#### 撤销/重做系统（Undo/Redo）
- **无限撤销**：记录每一步操作（包括旋转），支持 Ctrl+Z 撤销，Ctrl+Y 重做。
- **视觉反馈**：撤销时拼块带有"回退"动画，区别于正向移动。

---

### 2.3 智能辅助系统（解决"卡死挫败"问题）

#### 可移动高亮（Movable Highlight）
- **触发**：按H键（或点击提示按钮）。
- **表现**：所有与空白块相邻的拼块边缘发出呼吸灯式金色脉冲，持续3秒。
- **限制**：每次使用提示增加5步"惩罚步数"（仅影响排行榜，不影响实际步数）。

#### 下一步最佳提示（Next Best Hint）
- **触发**：按G键。
- **算法**：基于曼哈顿距离启发式，计算当前状态下哪个可移动拼块的移动能让"总距离和"减少最多。
- **表现**：目标拼块出现箭头指引，并显示虚线轨迹。

#### 实时预览窗（Live Preview）
- **位置**：游戏界面右上角，可拖动。
- **内容**：显示完整原图缩略图，当前拼块位置用半透明红色框标注。
- **交互**：鼠标悬停在预览窗的某个区域时，主游戏区对应拼块高亮。

---

### 2.4 视觉与特效系统（解决"反馈薄弱"问题）

#### 粒子特效系统（Particle System）
- **移动拖尾**：拖拽拼块时产生半透明残影，增强速度感。
- **交换闪光**：拼块交换位置瞬间产生白色星尘爆发。
- **胜利烟花**：完成拼图时从空白块位置发射多色粒子，呈抛物线散落，持续4秒。
- **旋转波纹**：旋转拼块时产生螺旋状扩散波纹。

#### 完成庆祝动画（Victory Animation）
- **阶段1**：所有拼块依次翻转显示完整原图（翻牌效果）。
- **阶段2**：原图中央弹出「完美！」字样，带弹性缩放动画。
- **阶段3**：显示统计面板（时间、步数、评级S/A/B/C），并播放对应音效。

#### 动态背景（Dynamic Background）
- 菜单界面使用缓慢飘动的抽象色块作为背景，避免静态空洞感。
- 游戏内背景根据当前拼图主题色调自动适配。

---

### 2.5 挑战与竞技系统（解决"重复可玩性"问题）

#### 限时挑战（Time Attack）
- **规则**：倒计时60秒（难度越高时间越宽裕，3×3=60s, 4×4=120s, 5×5=180s）。
- **奖励**：在时限内完成获得时间奖励分 = 剩余秒数 × 10。
- **失败处理**：时间到则游戏结束，可选择「再试一次」或「放弃」。

#### 限步挑战（Step Limit）
- **规则**：限制最大步数（3×3=20步, 4×4=40步, 5×5=80步）。
- **机制**：步数耗尽前必须完成，否则失败。
- **策略性**：迫使玩家规划路径，而非盲目点击。

#### 本地排行榜（Local Leaderboard）
- **维度**：按难度（3×3/4×4/5×5/旋转模式）分别记录：
  - 最快完成时间（Top 10）
  - 最少步数（Top 10）
  - 最高挑战分数（Top 10）
- **存储**：本地JSON文件，支持导出为文本。

---

### 2.6 音效与氛围系统

#### 分层音效
| 事件 | 音效特征 |
|-----|---------|
| 拼块移动 | 木质"咔哒"轻响，带轻微混响 |
| 拼块旋转 | 机械齿轮转动声 |
| 提示使用 | 风铃清脆声 |
| 撤销操作 | 磁带倒带"嗖"声 |
| 倒计时警告 | 最后10秒心跳声加速 |
| 拼图完成 | 管弦乐和弦+掌声 |
| 解锁新图 | 宝箱开启+金币声 |

#### 背景音乐（BGM）
- 菜单：轻快的钢琴曲循环。
- 游戏中：根据主题切换氛围音乐（自然/古典/电子）。
- 完成时：音乐升调+欢呼采样。

---

### 2.7 社交与分享功能

#### 成绩卡片生成（Score Card）
- 完成拼图后自动生成一张640×360像素的分享卡片。
- 包含：原图缩略图、难度标识、完成时间、步数、评级、日期水印。
- 保存到 `screenshots/` 目录，提示玩家分享路径。

## 三、技术实现要点

### 3.1 状态机设计（State Machine）
游戏采用分层状态机管理复杂流程：
```
AppState
├── MENU          # 主菜单
│   ├── SUB_PLAY      # 选择图鉴/难度
│   ├── SUB_COLLECTION # 图鉴浏览
│   ├── SUB_RANK      # 排行榜
│   └── SUB_SETTINGS  # 设置
├── PLAYING       # 游戏进行中
│   ├── SUB_NORMAL    # 普通模式
│   ├── SUB_TIME_ATTACK # 限时挑战
│   └── SUB_STEP_LIMIT  # 限步挑战
├── PAUSED        # 暂停
├── VICTORY       # 胜利结算
└── GAME_OVER     # 挑战失败
```

### 3.2 拼块旋转的数据结构
每个拼块不再只是 `(row, col)` 坐标，而是 `(row, col, rotation)` 三元组：
- `rotation ∈ {0, 90, 180, 270}`
- 渲染时使用 `pygame.transform.rotate()`
- 正确性判断：`current == target AND rotation == 0`

### 3.3 可解性保证（Solvability Guarantee）
打乱算法必须保证拼图有解：
- **无旋转模式**：从完成状态开始，模拟随机有效移动1000次，确保可达。
- **旋转模式**：先保证位置可解，再随机给每个拼块附加旋转（目标拼块除外）。

### 3.4 存档格式（JSON）
```json
{
  "version": "1.0",
  "settings": {
    "difficulty": "4x4",
    "control_mode": "drag",
    "sound_enabled": true,
    "rotation_mode": false
  },
  "collection": {
    "unlocked": [0, 1, 2],
    "completed_count": [1, 0, 0]
  },
  "records": {
    "3x3": {"best_time": 45.2, "best_steps": 18},
    "4x4": {"best_time": null, "best_steps": null}
  },
  "achievements": ["first_try", "step_master"]
}
```

### 3.5 性能优化
- 拼块图片预先裁剪并缓存，避免每帧裁剪。
- 粒子使用对象池（Object Pool）复用，避免频繁创建/销毁。
- 仅重绘变化区域（Dirty Rect），静态背景不重复渲染。

## 五、与基础要求的对照表

| 基础要求 | 本方案实现 | 升级点 |
|---------|-----------|--------|
| 游戏初始化与窗口 | 全屏/窗口自适应，动态分辨率 | 支持缩放、多分辨率适配 |
| 图像处理与拼块生成 | PIL裁剪 + 旋转支持 | 增加旋转状态、缓存优化 |
| 拼块随机打乱 | 模拟随机移动保证可解 | 可解性数学保证，非纯随机 |
| 用户交互处理 | 点击 + 拖拽双模式 | 更符合现代操作直觉 |
| 游戏完成判断 | 坐标+角度双重校验 | 支持旋转模式 |
| 原图提示 | 实时可拖动预览窗 | 带位置映射高亮 |
| 难度分级 | 3×3/4×4/5×5 + 旋转开关 |  exponentially 难度曲线 |
| 计步功能 | 基础步数 + 惩罚步数 | 区分实际步数与提示代价 |
| 计时功能 | 正计时 + 倒计时双模式 | 支持挑战模式 |
| 音效 | 8种事件音效 + BGM | 分层混音、无文件时合成 |
| 重新开局 | 当前窗口一键重开 | 保留设置，秒开新局 |

## 六、创新点总结

1. **旋转地狱模式**：将拼图从排列组合问题升级为排列+方向双重谜题，硬核玩家专属。
2. **图鉴进度系统**：赋予拼图"收集养成"属性，大幅提升重复打开率。
3. **智能提示算法**：基于曼哈顿距离的启发式提示，既帮助新手又不直接给答案。
4. **成绩卡片生成**：让单机游戏具备社交传播属性。
5. **操作模式自由切换**：兼顾传统玩家与现代玩家习惯。
