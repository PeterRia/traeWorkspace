# 归档包清单

本归档包用于把“好玩的拼图游戏设计”一次性交给 Codex 实现。建议按以下顺序阅读：

1. `00_一键交给Codex的启动提示词.md`：直接复制给 Codex 的完整提示词。
2. `01_功能范围与需求规格说明.md`：明确哪些功能必须做、哪些功能可以降级。
3. `02_技术架构与目录规范.md`：规定项目结构、模块职责、算法和路径处理。
4. `03_开发流程与任务拆解.md`：让 Codex 按阶段实现并自检。
5. `04_校验验收与测试清单.md`：最终判断项目是否可交的检查表。
6. `05_打包交付与归档规范.md`：说明如何生成 exe 和 `学号姓名.rar`。
7. `06_玩家使用手册模板.md`：Codex 生成项目时可直接改成用户手册。
8. `07_README模板.md`：Codex 生成项目时可直接改成 README。
9. `08_Codex收尾报告模板.md`：要求 Codex 完成后按模板汇报结果。
10. `09_原始设计文档_整理版.md`：保留原始扩充设计，方便追溯。
11. `10_一页版验收清单.txt`：提交前快速核对。
12. `拼图游戏Codex一次性交付总文档.docx/pdf`：整合版说明书，适合阅读或打印。

建议你实际使用时：

- 先把 `00_一键交给Codex的启动提示词.md` 复制给 Codex。
- Codex 完成后，用 `04_校验验收与测试清单.md` 检查。
- 最后按 `05_打包交付与归档规范.md` 压缩提交。
